# おりがみナビ(プロトタイプ)

折り紙の折り方を3Dで順番にナビゲーションするスマホ向けWebアプリのプロトタイプ。
「折り紙版Google Maps」— 現在の工程・次の折り・完成までの残り工程を常に表示する。

## 起動

```bash
npm install
npm run dev
```

## 現在できること

- 作品ライブラリ(チューリップ・犬・コップ・ことり・正方基本形・鶴・手裏剣・ねこのかお・きつねのかお・うさぎのかお・ぱんだのかお・くじらの12作品+準備中プレースホルダ)。
  サムネイルは工程データを最後まで適用した完成形をSVGで自動描画
  ([src/CreasePattern.tsx](src/CreasePattern.tsx))
- 工程データ駆動の3D折りシミュレーション
  (谷折り=シアン/山折り=ローズ/中割り折り=琥珀/かぶせ折り=菫)
  - 折り線の点線表示・折る面のハイライト・折る方向の3D矢印
  - 紙の表裏色分け(表=薄い赤/裏=白)
- 大人向けのダークUI(墨色ベース+朱アクセント、見出しは Shippori Mincho、
  英字は Space Grotesk、全面に紙質ノイズ)

  - 「手前の1枚だけ折る」などレイヤーを区別した折り

## アーキテクチャ

作品は動画や画像ではなく**工程データ**として持つ([src/engine/types.ts](src/engine/types.ts))。

- `vertices`: 展開図(平らな紙)上の2D座標。すべての折り線の端点を事前に埋め込む
- `faces`: 折り線で分割された面(多角形)
- `steps[]`: 1工程=複数の折り `folds[]`(折り線の頂点ペア+回転する頂点集合+山/谷+角度)
  +説明文。左右対称の折りは1工程に2本入れて同時に折れる(チューリップ工程2)

3D形状は [src/engine/fold.ts](src/engine/fold.ts) の `computeFoldState(model, t)` が
タイムライン位置 t(0=展開状態、k=工程kまで完了)から毎フレーム計算する。
工程を順に適用し、折り線の現在位置を軸に対象頂点をクォータニオン回転させる。
谷折り/山折りの回転方向は「谷=手前(+z)へ動く」から自動決定。
180°ちょうどではなく170°前後で折ることで、紙の層の重なり(Zファイティング)を回避している。

描画は [src/three/PaperScene.ts](src/three/PaperScene.ts)(three.js、React非依存)。
表と裏を別メッシュ(FrontSide/BackSide)で色分けし、面ごとの頂点色でハイライトする。

## 作品の追加方法

[src/models/dog.ts](src/models/dog.ts) を参考に:

1. 展開図に折り線をすべて書き、交点・端点を `vertices` に列挙する
2. 折り線で区切られた面を `faces` に列挙する(表から見て反時計回り)
3. 工程を順に `steps` に書く。各折りは折り線上の2頂点を `axis`、動く側の頂点を
   `moving` に指定。左右対称に同時に折るなら `folds` に2本入れる
4. `src/App.tsx` の `MODELS` に追加

注意点:
- 後の工程で使う折り線も最初から `vertices`/`faces` に埋め込む(面の動的分割はしない)。
  折り線が面を横切る場合は必ずその線で面を分割しておくこと(斜めに曲がる欠陥が出る)
- 折り先が辺や折り線に「ぴったり重なる」座標は厳密に計算する
  (コップの s=2-√2 参照。近似だとはみ出しが目視でわかる)
- 同一工程内の複数折りは、互いの `axis` 頂点を動かさないこと

## 中割り折り・かぶせ折りの表現

ほぼ平らに畳まれた多層フラップの中割り折り(inside reverse)は、
「折り線を軸に対象領域の全レイヤーを約180°回転」で表現している
(レイヤーの前後が入れ替わるため、見える紙の色も実物と一致する)。
`FoldOp.type` に `'inside-reverse' | 'outside-reverse'` を指定し、
`sweep: 'front' | 'back'` で回す向きを選べる。「ことり」
([src/models/bird.ts](src/models/bird.ts))が実装例。
後工程の折り線は「完成状態で決めた線を、先行する折りで鏡映して展開図に埋め込む」
(bird.ts のコメント参照)。

## 工程データ作成ツール(β)

ライブラリ画面フッターの「MODEL EDITOR」から起動([src/Editor.tsx](src/Editor.tsx))。
「新規(正方形)」から作品を一から作れる。

- **折り線ツール**:展開図上で2点をクリック(頂点スナップ+数値微調整可)
  → その直線で全ての面を自動分割し、辺との交点に頂点を自動追加
  ([src/engine/split.ts](src/engine/split.ts))。既存頂点番号は保持される
- **工程ビルダー**:頂点クリックで「軸(2頂点)」「動く頂点」を指定し、
  折り種類(谷/山/中割り/かぶせ)・角度・回す向きを設定して工程を追加。
  1工程に複数の折り(同時折り)も組める。既存工程の削除・並べ替え対応
- **JSONタブ**:UIでの編集内容と双方向同期。直接編集して反映(検証つき)、
  コピーしてモデルファイル(src/models/)に貼り付けられる
- **3Dライブプレビュー**:タイムラインスライダーで任意の時点を確認
- 元に戻す(30段階)、既存4作品のプリセット読込

## 基本形のたたみ込み(単一頂点平坦折り)

正方基本形(square base)は「折り線(中心-頂点)を軸にした連鎖回転6回」で表現
([src/models/squareBase.ts](src/models/squareBase.ts))。厳密±180°では回転符号に
よらず同一の平坦形に畳まれ(tools/solve-collapse.mjs で検証)、符号は層の重なり順
(アコーディオンの向き)にだけ効くため、176°+交互方向を採用。
`FoldOp.direction` で回転符号を明示できる(山谷の自動判定のオーバーライド)。
折りが4本以上の工程は3D矢印を省略し折り線のみ表示、バッジは「たたむ COLLAPSE」。

## 実装メモ:鶴

鶴は Web Japan Kids Web の鶴ページを基準にした14工程で実装済み。流れは、三角に折る → さらに半分 → ふくろを開いて正方形 → 反対側も正方形 → 花弁折り → 細く折る → 首・尾・頭の中割り折り → 羽を広げる、の順番。
参照元: https://web-japan.org/kidsweb/ja/virtual/origami/exploring02f.html。花弁折りの厳密点は `tools/solve-crane-petal.mjs` で検算できる。

注意:シート座標は -112.5° 回転済み(squareBase.ts コメント参照)。
新しい折り線の座標はこの回転後の座標系で与えること。

## 2枚組み(モジュラー)の表現

手裏剣([src/models/shuriken.ts](src/models/shuriken.ts))はおりがみくらぶの折り図に忠実な
全9工程(観音折り→半分→端を斜め45°→端の帯を折り返して翼に→朱を裏返して90°回し中央へ→
藍を上にかさねる→差し込む→裏返す→差し込む)。展開図は0.5刻みの5×5格子+対角線の折り線で、
❹の折り線は❸の斜め辺と平行(ユニットは翼が斜めにずれた六角Z形、星の尖りは風車状に傾く)。
正方形2枚(朱=鏡映・藍)を同一 `vertices`/`faces` に左右へ並べて同居させ、各シートを既存の
回転エンジンで折ってから組み立てる。幾何は `tools/solve-shuriken.mjs` で厳密値を検証済み。
「さしこむ」は新しい折り線を作らず、❹の折り線で翼を開いて相手の帯を巻き込み反対側へ
閉じ直す(角度344°の1回転オペ)。組み立て用のエンジン拡張: `FoldOp.translate`(剛体平行移動)、
`FoldOp.spinZ`(面内回転。「裏返して向きをかえる」を裏返し180°+面内90°+移動の1オペで表現)、
`FoldType 'assemble'`。シート別の色分けは `OrigamiModel.faceSheet`+`sheetColors`。
平らな作品は `cameraPos` で正面寄りから見せる。

多層の平畳みで層の重なり順を正しく見せるための技法(shuriken.ts 参照):
- 折りの残差角(180°との差)を工程が進むほど大きくして層を扇状に分離する
- 折り線上に来た紙のはしは以降の折りの moving に含めない(軸上の点が回転すると紙面裏へ潜る)
- 折り目の縁が非隣接の層で共有される場合(❷の中心線、❹の折り線)は頂点を層ごとに複製し、
  上側を z 補正で持ち上げる。z 補正は大きな回転が全部終わったあとに行う
  (回転前に持ち上げると軸から浮いた蝶番になり面がねじれる)

## 既知の制約
- 差し込みは翼の1回転オペ(開いて→巻き込んで閉じる)による表現。層内部の
  重なり順は残差角に依存し、完成形の中央にわずかな白のぞきが残ることがある
- かぶせ折りは色の入れ替わり(実物では裏の色が出る)を再現しない
- エディタはJSON直接編集+面分割(折り線の対話的作図UIは一部のみ)
- スワイプで次へ/戻る は未実装(3D回転ドラッグと競合するため保留)
- 工程ごとの推奨カメラ角度は未実装
